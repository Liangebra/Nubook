## NuBook を選ぶ理由（市販の主流家計簿アプリとの比較）

市販の家計簿アプリは、広告やソーシャル機能、強制のクラウド同期制限などで肥大化しがちです。単に個人の収支を記録したいだけの人にとって、NuBook は次の 4 つのコアな側面で究極の削ぎ落としと根本的な革新を実現しました。

### 1. 絶対的なデータ主権と多層的なフォールトトレラントなインポート
*   **競合アプリの課題**: データの人質化。多くのアプリは家計簿をアプリのサンドボックス内に暗号化して保存し、解析が難しい独自フォーマットでエクスポートさせたり、「Excel/CSV エクスポート」を有料 VIP 機能として課金したりします。
*   **NuBook の革新**: データの 100% 自由な流通。ワンタップで JSON 抽象ツリー辞書、JSONL 行指向、または人間が読める標準 CSV テーブルを瞬時に生成し、システムの公開ディレクトリ `Documents/NuBook` に直接保存できます。さらに強力なのは、インポートモジュールに実装した **三段階インテリジェント検出システム** です。使用例：エクスポートした CSV をパソコン上で自由にフィールドを変更したり、メモ帳で JSON と混ぜてスマートフォンに戻したりしても、システムエンジンは破損した行を自動的に遮断し、規範的なデータを強制的に抽出して Room データベースに書き込み、復元することができます。

### 2. 即時代数解析、ポップアップ計算機は不要
*   **競合アプリの課題**: 「野菜 380 円、ペットボトル飲料 2 本で 400 円、さらに店舗ポイント還元 50 円を引く」といった計算をする場合、いちいち計算機アプリを起動して計算し、結果をコピーして入力画面に貼り付ける必要があります。
*   **NuBook の革新**: 入力は数字に限定されません。入力レイヤーは強力な `mXparser` ライブラリと連携しています。使用例：上記のような金額の断片が頭に浮かんだら、入力欄に `380+(200*2)-50` と自由に入力するだけで、システムがリアルタイムで抽象構文木 (AST) に分解し、浮動小数点の結果を解析して保存します。上級ユーザー向けの高度な数値入力体験をお楽しみください。

### 3. ピクセル単位の深いテーマ適応、テンプレートからの脱却
*   **競合アプリの課題**: インターフェースは「ライトモード／ダークモード」の選択のみ、またはスキンの購入が必要です。スキンを適用しても、未対応の部分や重厚なドロップシャドウが残ることがよくあります。
*   **NuBook の革新**: 究極のフラットデザイン哲学（アプリ全体でカードの Z 軸シャドウを排除し、グローバルに `elevation="0dp"` を適用）に加え、グローバルに浸透する `ColorEngine` エンジンを採用しています。ユーザーが選んだメインカラーに基づき、システムはライフサイクルを介して、画面下部のナビゲーションバーの 3 つのアイコン、家計簿リストの各レコード左側の柔らかなマスク帯、さらには統計グラフの円グラフのセクターに至るまで、すべてピクセル単位でそのメインカラーに動的に再着色します。シームレスに融合した没入感あふれる美しさを体感できます。

### 4. データファイアウォール：バキューム実行環境
*   **競合アプリの課題**: 登録に電話番号が必要、バックグラウンドで常駐して金融商品の勧誘や位置情報・写真へのアクセスを行うなど、プライバシーへの配慮が不足しています。
*   **NuBook の革新**: 完全な無干渉とバキューム実行メカニズム。バックグラウンド常駐やアラーム管理（AlarmManager）を完全に排除しているだけでなく、アプリの設定ファイル `AndroidManifest.xml` において、インターネット（INTERNET）権限すら一行も要求していません。ネットワーク接続が存在しないため、NuBook はローカルの SQLite（Room）とのみ安全に連携します。ソーシャル機能もスプラッシュ画面もありません。ここは数字を記録するだけの場所です。

## 技術スタックとアーキテクチャ

本プロジェクトは Google が推奨する最新の Android アーキテクチャの規範に準拠しています。
- 開発言語：Kotlin 1.9+
- 非同期フレームワーク：Kotlin Coroutines + Flow
- ライフサイクル：Jetpack Lifecycle, ViewModel, LiveData
- データ永続化：Jetpack Room
- UI コンポーネントライブラリ：Material Components for Android（軽度カスタマイズ版）
- グラフ描画：MPAndroidChart（シャドウ除去、ソフトカラーフィル適用）
- 依存関係管理／ビルドエンジン：Gradle (KTS) / AGP 8.2.2

## プロジェクト構造ガイド

```text
com.nubook/
├── NuBookApplication.kt  # グローバルアプリケーションエントリ宣言
├── data/
│   ├── export/       # 三段階インテリジェントデータインポート／マルチプロトコルエクスポート解析エンジン
│   └── local/        # Room Database、Entity テーブル構造、Dao 検索インターフェース
├── domain/           
│   └── usecase/      # クリーンなビジネスロジック処理ユニット（コア計算、フィルタリング、統計集計など）
└── ui/
    ├── base/         # BaseActivity（テーマレベルオーバーライドエンジンを提供）
    ├── home/         # ホーム集約パネルと複数家計簿フィード
    ├── input/        # 記帳入力画面と mXparser 数式エンジンブリッジ
    ├── ledger/       # 特定の家計簿の詳細履歴
    ├── search/       # フローベースのグローバルキーワードリアルタイムフィルタリング
    ├── settings/     # カラーエンジン切り替えのメインエントリ
    ├── statistics/   # MPAndroidChart グラフ表示ロジック
    └── theme/        # ColorEngine コアレンダラー
```

## ビルドと実行

環境要件：JDK 17+、Android Studio Iguana / Jellyfish（またはそれ以降）

本リポジトリをクローン、ZIP ダウンロード、またはローカルにプルします。

Android Studio でプロジェクトを開きます。

Android SDK が未設定の場合、local.properties ファイルに必要な設定が生成されます。

Gradle Sync が完了し依存関係の解決が終わるのを待ち、Run 'app'（緑色の三角ボタン）をクリックして、実機またはエミュレータ（対象 Android API >= 24）でビルド・実行します。
